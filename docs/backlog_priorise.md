# Backlog priorisé (proposition)

## Légende
- **P0** : indispensable au MVP strict
- **P1** : MVP étendu / proche du cœur
- **P2** : ultérieur

## P0 — MVP strict (end-to-end)

1. **Auth basique (email + mot de passe)**
   - Critères : inscription, connexion, déconnexion.
2. **Parcours simple**
   - Critères : création d’un parcours avec étapes, affichage du parcours.
3. **Questionnaire par étape**
   - Critères : quiz simple, scoring, résultat visible.
4. **Progression utilisateur**
   - Critères : statut “en cours” / “complété”.
5. **Certificat basique**
   - Critères : génération automatique si score ≥ seuil.

## P1 — MVP étendu (proche du cœur)

6. **Reset mot de passe**
7. **Parcours multiples + tags**
8. **Banque de questions**
9. **Feedback détaillé (explications par question)**
10. **Certificat partageable (lien public)**
11. **Reporting simple (taux de complétion)**

## P1 — Extension : Labs interactifs intégrés

12. **Lancement de lab depuis une étape de parcours**
    - Critères : bouton “Lancer le lab”, création de session.
13. **Provisioning d’environnement isolé**
    - Critères : container/sandbox par utilisateur, quotas.
14. **Connexion aux outils internes**
    - Critères : accès limité aux outils nécessaires (API / CLI). 
15. **Retour de résultat dans la plateforme**
    - Critères : statut réussite/échec, logs minimalistes.

## P2 — Futur

16. **SSO avancé**
17. **Certificats enrichis (QR, badges)**
18. **Analytics avancés**
19. **Marketplace de labs**
20. **Lab multi-utilisateurs**
