# Guide UI des états

Ce guide définit les quatre états standards utilisés dans l'interface.

## États

### Vide
- **Usage**: aucune donnée disponible.
- **Ton**: positif, incitatif.
- **Éléments**: illustration simple, titre, description courte, action principale.

### Loading
- **Usage**: données en cours de chargement.
- **Ton**: neutre, informatif.
- **Éléments**: skeletons ou placeholders alignés à la structure finale.

### Erreur
- **Usage**: échec de récupération ou d'action utilisateur.
- **Ton**: clair, rassurant.
- **Éléments**: titre explicite, message, action de reprise (retry) ou alternative.

### Succès
- **Usage**: données chargées et affichées.
- **Ton**: neutre, factuel.
- **Éléments**: contenu principal, actions secondaires si nécessaire.

## Règles communes
- Les états doivent occuper la même zone que le contenu final.
- Les actions proposées doivent être évidentes et ne pas multiplier les choix.
- Prioriser la clarté du message et la cohérence visuelle.
