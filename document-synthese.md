# Document synthétique — Plateforme SkillForge

## Objectifs
- Offrir un parcours clair pour découvrir, s’inscrire et suivre une formation.
- Simplifier la création et la publication de contenus pour les formateurs.
- Assurer un suivi de progression et une validation des acquis.

## Parcours (prioritaires)
1. **Apprenant** : arrivée sur la plateforme → découverte d’un cours → inscription → consommation du contenu → validation/attestation.
2. **Formateur** : création de compte → création d’un cours → publication → suivi des apprenants.
3. **Administrateur** : validation des cours → gestion des utilisateurs → reporting.

## Fonctionnalités MVP
- Authentification (inscription/connexion) pour apprenants et formateurs.
- Catalogue de cours avec recherche/filtre basique.
- Page détail cours + inscription.
- Lecture de contenu (vidéo/texte) + suivi de progression.
- Tableau de bord formateur (création/édition/publication de cours).
- Tableau de bord admin (validation des cours, gestion utilisateurs).

## Contraintes
- **Temps** : MVP livré en cycles courts (ex. 8–12 semaines).
- **Budget** : prioriser les fonctionnalités cœur (apprentissage et publication).
- **Sécurité** : protection des données personnelles (RGPD) et contrôle d’accès.
- **Qualité** : UX simple et stable sur desktop/mobile.

## Schéma simple des parcours prioritaires
```
Apprenant : Arrivée → Découverte → Inscription → Consommation → Attestation
Formateur : Inscription → Création cours → Publication → Suivi apprenants
Admin     : Validation cours → Gestion utilisateurs → Reporting
```

## Définition du succès (KPIs)
- Taux de conversion visite → inscription ≥ 10%.
- Taux d’activation (inscrits qui commencent un cours) ≥ 60%.
- Taux de complétion d’un cours ≥ 40%.
- Temps moyen avant première leçon ≤ 24h.
- Satisfaction (CSAT) ≥ 4/5.
